# Codex Task Rules

- Make small, focused changes.
- Do not rewrite the whole project without a clear reason.
- After each task, report the list of changed files.
- Do not add dependencies without explaining why they are needed.
- Do not build a complex system without an intermediate test.
- Check the build after each development stage when local Bannerlord DLLs are available.
- Keep the first-load path stable and safe.
- Prefer read-only observation and logging before changing campaign AI behavior.
- Do not commit game DLLs or generated archives.
